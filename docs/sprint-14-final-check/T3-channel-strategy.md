# T3 — Launch Channel Strategy

## Goal

Create `docs/launch/CHANNEL-STRATEGY.md` — a realistic launch channel plan that accounts for Josh's actual audience distribution.

## Context

Josh has:
- **Strong existing audience** on Facebook and Instagram (personal/professional network, Nashville Chopin community, Unagi contacts)
- **Zero audience** on Hacker News, X/Twitter, Medium, dev.to
- **Three repos** on GitHub (termdeck, mnestra, rumen) with zero stars

The launch collateral (Show HN post, Twitter thread, LinkedIn, dev.to) was written assuming HN as the primary channel. But cold-launching on HN with zero karma, zero followers, and no pre-seeded comments is the single biggest failure mode for zero-audience authors (per LAUNCH-STATUS-2026-04-15.md).

## Content

Write a channel strategy (100-150 lines) covering:

### Leverage existing reach first
- Facebook post (Josh's actual network sees it, some are developers)
- Instagram story/reel with the GIF (visual hook, different audience)
- Direct DMs to developer friends (already started with David, Jonathan, Yasin)
- LinkedIn post (professional network)

### Bootstrap new channels
- **Hacker News**: Create account now if not existing, post a few genuine comments on devtool posts to build karma before Show HN. Show HN timing: Tuesday/Wednesday 8am PT. Pre-seed with 3-5 friends who will provide early substantive comments (not fake upvotes — real feedback).
- **X/Twitter**: Post the thread, but don't expect organic reach from zero followers. Cross-link from FB/IG/LinkedIn. The GIF is the viral asset.
- **dev.to**: Cross-post the 4+1 blog post. dev.to's algorithm favors long-form "I built this" stories regardless of follower count.
- **Medium**: Skip for launch. Medium requires membership and penalizes external links. dev.to is strictly better for devtools.
- **Reddit r/commandline, r/selfhosted**: Good fit for "tmux in the browser" angle. Post after Show HN, reference the HN discussion.

### Timing sequence
Hour-by-hour launch day plan.

### What NOT to do
- Don't post on r/programming (anti-self-promo rules)
- Don't buy upvotes or fake engagement
- Don't post on Product Hunt yet (needs a landing page, not just a GitHub repo)
- Don't spread across too many channels simultaneously — focus effort on 2-3 that can compound
