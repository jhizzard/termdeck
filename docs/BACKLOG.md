Two compatible interpretations:
  - **Path A — render-as-one-line (visual).** When a multi-line text block is pasted into a TermDeck terminal (via the dashboard input box OR via cross-panel paste-from-output), the rendered visual collapses internal line breaks into spaces, displaying as one line. Doesn't alter the underlying PTY input — the agent receives the original multi-line text — but the operator sees a clean single line. Cheaper, doesn't touch semantics. Surface: client-side render hook in `packages/client/public/app.js` around the `clipboard.copy` / `clipboard.paste` event flow + xterm.js paste handling.

Replace mid-paragraph `\n` with spaces (preserving real paragraph breaks); collapse runs of `\n\n+` into single `\n\n`; strip trailing whitespace on each line. Surface: server-side `normalizePasteShape(text)` helper invoked from the input route + client-side mirror for cross-panel paste.

Verify at scoping by examining one of Joshua's actual broken-paste blocks via DevTools clipboard inspection.
  - **Acceptance.** From a TermDeck panel running a wide-output command, copy a wrapped line, paste into another TermDeck panel's input box.
