The system already detects problems (Flashback's detect‚Üírecall‚Üísurface chain works end-to-end) and already compiles incidents into prose rules, but the loop is open at three edges: recall results never reach a working agent's context, nothing measures whether any of the 7,567 stored memories is ever used, and recurring kitchen lessons pile up as 4‚Äì8 near-duplicate rows instead of becoming ratified, distributable doctrine. Sprint 78 closes the three highest-leverage edges with the least new machinery: a machine-readable doctrine registry (the substrate every later stage consumes), a live-advisory MVP that re-routes the existing error trigger into the agent's PTY/context instead of a browser toast, and a recall-hit telemetry log that makes every future pruning/elevation decision data-driven. Sprint 79 then builds the elevation pipeline (Rumen doctrine-scan ‚Üí Haiku draft ‚Üí PR ‚Üí ratify ‚Üí flow-back as a recallable `doctrine` row) and the capture gates (non-destructive reinforcement counting, provenance on the MCP path, granularity classification) that feed it clean signal. All four stage designs survive their adversarial verdicts, but only as amended ‚Äî two verdicts returned holds:false on the literal text, and every blocking amendment is folded into the lane scopes below as constitutive, not optional. The unifying synthesis decision: there is ONE registry (repo `doctrine/registry.jsonl` + a never-shipped `~/.claude/doctrine/registry.local.jsonl` overlay), consumed by git hooks, the advisor, and sprint-inject alike, with the Rumen-side `doctrine_registry` DB table acting only as the elevation pipeline's staging state that materializes INTO the repo registry at ratification.

## 2. Ground truth ‚Äî today's gaps (reader evidence)

### Gap 1 ‚Äî No mid-task recall ever reaches an agent (delivery gap, not detection gap)
Flashback proves detect‚Üírecall‚Üísurface end-to-end: `session.js::_detectErrors` runs on every PTY chunk (adapter regexes + `PATTERNS.shellError`, 30s rate limit at session.js:~487), `onErrorDetected` ‚Üí `mnestraBridge.queryMnestra` ‚Üí `pickNextNonDismissed` ‚Üí SQLite `flashback_events` ‚Üí WS frame (index.js:2134-2237). But the frame terminates in `showProactiveToast` for the HUMAN (packages/client/public/app.js:1023); **no path writes a recall result into any agent's PTY or context** ‚Äî index.js:2196-2198 sends only `sess.ws.send(frame)`, never `session.pty.write`. Every agent-side recall is voluntary (CLAUDE.md mandate or injected boot-prompt text); ~/.claude/settings.json wires only SessionEnd + PreCompact ‚Äî **zero** SessionStart/PreToolUse/PostToolUse/Stop hooks, all of which support context injection per the live hooks docs. The delivery primitives already exist and are proven: `POST /api/sessions/:id/input` two-stage bracketed paste (sprint-inject.js, single-stage BANNED), `/poke` cr-flood, idle detection via the buffer status field.

### Gap 2 ‚Äî Zero recall telemetry: "never recalled" is uncomputable
memory_items has 21 columns and none is a usage counter; `mnestra_commands` = 0 rows; `mnestra_recent_activity` = 0 rows; no recall/usage log table exists in the 92-table public schema. All 7,567 active memories are indistinguishable on read-side value ‚Äî every capture-quality claim is unfalsifiable, and pruning/decay/elevation thresholds are calibrated from three hand-probed examples. memory_relationships (2,959 rows) shows graph-linking IS instrumented; recall usage is the only un-instrumented surface. ~2 weeks of logging at current recall volume separates live core from dead weight.

### Gap 3 ‚Äî No repetition detection; the store re-learns instead of reinforcing
The dedup that exists is destructive (remember.ts:100-133: the 0.88‚Äì0.95 band REPLACES content+embedding+metadata wholesale), project-scoped (same kitchen lesson coexists per-project forever), silently disabled on RPC error (remember.ts:96-98), and leaky (56 exact content_hash dup groups ‚Äî and note: no content_hash COLUMN exists; that figure came from ad-hoc profiling). Live probes: ~8 decision rows restate the auditor-checkpoint principle (5 written on 2026-05-21 alone), ~7 restate CPU-liveness ‚Äî one row literally self-reports the rule is "in Mnestra >=5x". Rumen can't help: `rumen_insights` has no embedding column, no de-dupe constraint (the code says so at src/index.ts:131-132), and ‚Äî contradicting its own README ‚Äî **nothing ever writes insights back to memory_items** (surface.ts:49-68 is the only INSERT; MNESTRA-COMPATIBILITY.md:68 confirms), so `memory_recall` can never return an insight and Relate can never cite one. `rumen_questions` is schema'd, never written (`questionsGenerated: 0` hardcoded).

### Gap 4 ‚Äî Prose-only enforcement on a corpus whose own thesis is "hooks > vigilance"
Global CLAUDE.md:394 states the thesis; ~70% of what the docs demand is machine-checkable in principle (post grammar, checkpoint cadence, test-glob membership, tarball contents, file ownership, publish-before-push, the RLS SQL already written verbatim at CLAUDE.md:286-313) but only a minority is machine-checked. Demonstrated prose failures: post-shape drift (Sprint 51.7 ‚Äî fixed by hardening the CONSUMER, status-merger.js), auditor compaction/silent death (51.6/51.7/Maestro 5 ‚Äî 9h lost), DONE-periphery gap (ClaimGuard 19, Sprints 73/74 ‚Äî "most likely to fail next," enforcement still pure prose at sprint-75 PLANNING.md:103-108), spec-marked-as-landed (Sprint 74 T4-GROK), substrate substitution (Maestro 9), release-order violation (Sprint 35, RELEASE.md:102 admits the countermeasure is "three layers of redundancy" of prose), existence-not-behavior acceptance three sprints running (INSTALLER-PITFALLS ledger #16). The docs themselves drift between tiers (project CLAUDE.md still says "4+1" vs global :149 "accurate naming is 3+1+1").

### Gap 5 (compounding) ‚Äî Capture-side hygiene holes that poison everything downstream
The hook tier (memory-session-end.js:755-788, memory-pre-compact.js:142-184, server spawners index.js:303-418) bypasses memoryRemember entirely: raw REST inserts, no dedup, no stripPrivate (raw transcript tails ‚Äî env vars included ‚Äî ship to OpenAI and Supabase), double-fire on /exit-then-PTY-close, one overlapping snapshot row per 10-min tick (~50 rows per 3-hr 4-panel sprint), and a false-`'claude'` source_agent coercion (:733-753). The MCP memory_remember tool ‚Äî the most deliberate path ‚Äî cannot set metadata/source_agent/sprint_ref at all (mcp-server/index.ts:214-242), so 92.6% of rows carry NULL provenance. The :37778 webhook has **no auth and binds all interfaces** (webhook-server.ts:282-303, :379) ‚Äî on a second machine that is a remote memory-poisoning endpoint for the doctrine feedstock.

### 3.1 Doctrine registry + doc enforcement (verdict: holds:false ‚Üí holds as amended)

**Kept core:** the inversion ‚Äî a machine-readable registry as source of truth for rules, prose demoted to rendered/derived views; JSONL-not-YAML; `check.type='manual'` honesty entries so coverage stats can't lie; seed inventory of already-mechanized rules; one check implementation consumed by three trigger surfaces; warn-first severity; orchestrator as sole registry writer; full-file version stamp (never the 4KB-head stamp that already failed).

**Applied amendments:**
- **Registry split (AMEND-1, blocker):** Joshua-only rules (the scrub rule, htmlBody entities, anything whose pattern would trip ~/.gitleaks.toml) live in `~/.claude/doctrine/registry.local.jsonl` ‚Äî never in repo, never in tarball. The repo registry carries only a stub for the scrub rule (no pattern text). gitleaks allowlist is NOT widened.
- **No new frontmatter parser (AMEND-2, blocker):** extend the shipped `packages/server/src/sprint-frontmatter.js` (Sprint 47) ‚Äî preserve fail-soft-on-missing / hard-fail-on-malformed semantics and the existing `{tag, agent}` lanes shape; doctrine fields (`gates`, `acceptance_regex`, `post_shape`, `checkpoint_cadence_min`, `owns`, `watches`) are additive keys.
- **enforcement.max_severity per surface (AMEND-3):** git-hook / PreToolUse-deny / inject-refusal may carry `block`; server-monitor and inject-advisory are capped at `warn` and the registry validator rejects entries claiming otherwise. **Status-append rules are structurally advisory forever** (lanes write STATUS.md via direct tool calls; no interception point) ‚Äî the registry must never imply promotion that is architecturally impossible.
- **Sprint-inject refusal is the one real new block surface (AMEND-4):** refuse to fire on missing/malformed frontmatter, anchor-check divergence, or registry self-validation failure. Lanes never booting on a defective contract is real enforcement; everything mid-sprint is advisory by construction.
- **Rate-limiting lives in the registry stage (AMEND-5):** `doctrine/index.js::shouldNotify(rule_id, dedupe_key)` backed by doctrine_events ‚Äî per-rule cooldown 30 min default, hard 3-advisories/lane/hour budget, overflow routes to the orchestrator panel; default routing for ALL advisories is the orchestrator.
- **status-lint never mutates STATUS.md (AMEND-6):** append-only is inviolable; "downgrade" is an advisory tag, and lint validates against the lane's declared `post_shape`, NOT status-merger's tolerance regexes run in reject mode.
- **Telemetry damping (AMEND-7):** raw per-fire doctrine_events stay in SQLite; Mnestra gets daily per-rule aggregates only; periodic checks skip when the watched file's mtime/size is unchanged.
- **Render budget + never auto-write (AMEND-8/9):** derived CLAUDE.md blocks render a COMPACT form under a byte ceiling; `render --write` NEVER runs automatically against ~/.claude/CLAUDE.md (protects same-turn hand edits); SessionStart/SessionEnd run `--check` report-only.
- **Brad hardening (AMEND-10):** doctrine/ vendored into stack-installer assets with the paired-version check extended to it; Brad's registry is read-only, active audience:'all' entries only, baked at publish; loadDoctrine version-checks at every load; no CLAUDE.md render machinery on his path.
- **Fail-soft everywhere (AMEND-11):** recordGateEvent never throws into hook/commit/inject paths; rls-audit exits 0 SKIPPED when no DB configured; cadence/periphery checks no-op with logged warning on unreadable inputs.
- **Five gates, not four (AMEND-12):** rls-audit.js mechanizes ALL FIVE hygiene gates incl. the `pg_tables ... NOT rowsecurity` query and advisor lints 0011/0013.
- **Verified evidence, not pattern presence (AMEND-13):** FIX-LANDED claims verified by `git cat-file -e <sha>` / in-glob test existence; where verification is impossible the rule stays `check.type=manual`.
- **checkpoint-cadence is a detector, not an actor (AMEND-14):** emits to the orchestrator only, with the lane's buffer status attached so silent-death (idle+stale) is distinguishable from deep work (thinking+fresh).

**Rejected sub-mechanisms (and why):** new `doctrine/frontmatter.js` (parser collision with shipped sprint-frontmatter.js ‚Äî the lockstep-drift class reintroduced at the parser layer); shipping scrub patterns in any repo/tarball file (the scrub mechanism would BE the leak); severity promotion to block on status-append surfaces (no blocking mechanism exists ‚Äî a lying registry is worse than prose); status-merger-as-rejecter (inverts a tolerance layer's purpose); auto-mutating STATUS.md posts; auto `render --write` on the global CLAUDE.md; pattern-only FIX-LANDED evidence regex (Goodhart ‚Äî trains decorative shas).
