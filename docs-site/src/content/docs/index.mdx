## The stack

Three small, independent tools that compose into a durable memory system for
terminal-driven development. Each tier works on its own; together, they give you
a control room that remembers.

<CardGrid stagger>
  <Card title="TermDeck" icon="laptop">
    A browser-based terminal multiplexer. Real PTYs, rich metadata overlays,
    per-terminal theming, and flexible grid layouts. Think tmux with a
    control-room UI.

    [Read the TermDeck docs →](/termdeck/)
  </Card>

  <Card title="Engram" icon="seti:db">
    A long-term memory store for developer sessions. Hybrid search (keyword +
    semantic + recency), tiered decay, project affinity, and MCP tools so Claude
    can recall and remember across sessions.

    [Read the Engram docs →](/engram/)
  </Card>

  <Card title="Rumen" icon="random">
    An async learning layer that runs over Engram on a schedule. Extracts facts,
    relates them to prior memories, synthesises insights, and surfaces them to
    future sessions — all without blocking your prompt loop.

    [Read the Rumen docs →](/rumen/)
  </Card>
</CardGrid>

## How they connect

TermDeck captures what you do. Engram remembers it. Rumen learns from it.
